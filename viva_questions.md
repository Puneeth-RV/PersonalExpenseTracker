# 🎓 Viva Questions — Personal Expense Tracker

---

## 📌 Project-Specific Questions

**Q1. What is this project about?**
> It is a Personal Expense Tracker web app where a user can add an expense with a name and amount, view all expenses, delete an expense, and see the total amount spent.

**Q2. What is the tech stack used?**
> - Frontend: React.js
> - Backend: Node.js with Express.js
> - Database: MongoDB with Mongoose
> - HTTP Requests: Axios

**Q3. What are the API endpoints in your project?**
> - `GET /expenses` → Fetch all expenses
> - `POST /expenses` → Add a new expense
> - `DELETE /expenses/:id` → Delete an expense by ID

**Q4. What is the Expense schema?**
> ```js
> { name: String, amount: Number }
> ```

**Q5. On which port does the backend run?**
> Port 5000

**Q6. On which port does the frontend run?**
> Port 3000

**Q7. What database is used and where is it running?**
> MongoDB, running locally on `mongodb://localhost:27017/expensetracker`

## ⚛️ React.js Questions

**Q8. What is React.js?**
> React is a JavaScript library used to build user interfaces. It uses components to build the UI.

**Q9. What is a functional component?**
> A functional component is a simple JavaScript function that returns JSX (HTML-like code). Example: `function App() { return <div>Hello</div>; }`

**Q10. What is useState?**
> `useState` is a React hook used to store and update data inside a component. When the state changes, the component re-renders.

**Q11. What is useEffect?**
> `useEffect` is a React hook that runs code when the component loads or when a value changes. We use it to fetch expenses when the page first loads.

**Q12. Why is useEffect used in this project?**
> To call `fetchExpenses()` automatically when the page loads, so all expenses are displayed without the user having to do anything.

**Q13. What is JSX?**
> JSX stands for JavaScript XML. It allows us to write HTML inside JavaScript. React converts it to normal JavaScript.

**Q14. What is the use of `key` in the list?**
> The `key` prop helps React identify which item in a list has changed. We use `expense._id` as the key.

**Q15. What is Axios?**
> Axios is a library used to make HTTP requests (GET, POST, DELETE) from the frontend to the backend.

## 🟢 Node.js Questions

**Q16. What is Node.js?**
> Node.js is a JavaScript runtime that allows us to run JavaScript on the server side (backend), not just in the browser.

**Q17. What is npm?**
> npm stands for Node Package Manager. It is used to install libraries and packages needed for the project.

**Q18. What is `node server.js`?**
> It is the command to start the backend server using Node.js.

---

## 🚂 Express.js Questions

**Q19. What is Express.js?**
> Express.js is a framework for Node.js that makes it easy to create APIs and handle HTTP requests like GET, POST, and DELETE.

**Q20. What is middleware?**
> Middleware is a function that runs before the request reaches the route. In this project, `express.json()` and `cors()` are middlewares.

**Q21. What is `express.json()` used for?**
> It parses incoming JSON data from the request body so we can access it using `req.body`.

**Q22. What is CORS and why is it needed?**
> CORS stands for Cross-Origin Resource Sharing. It allows the frontend (port 3000) to communicate with the backend (port 5000). Without it, the browser blocks the request.

**Q23. What is `req.body`?**
> `req.body` contains the data sent by the client in a POST request. For example, `{ name: "Food", amount: 100 }`.

**Q24. What is `req.params`?**
> `req.params` contains the URL parameters. For example, in `/expenses/:id`, the `id` value is accessed using `req.params.id`.

## 🍃 MongoDB & Mongoose Questions

**Q25. What is MongoDB?**
> MongoDB is a NoSQL database that stores data in JSON-like documents instead of tables and rows.

**Q26. What is Mongoose?**
> Mongoose is a library that helps us connect to MongoDB and define schemas (structure) for our data in Node.js.

**Q27. What is a Schema in Mongoose?**
> A schema defines the structure of a document in MongoDB. In this project, the schema has `name` (String) and `amount` (Number).

**Q28. What is a Model in Mongoose?**
> A model is created from a schema. It is used to interact with the MongoDB collection — to save, find, and delete documents.

**Q29. What does `Expense.find()` do?**
> It fetches all expense documents from the MongoDB collection and returns them.

**Q30. What does `Expense.findByIdAndDelete(id)` do?**
> It finds an expense by its `_id` and deletes it from the database.

**Q31. What does `new Expense({...}).save()` do?**
> It creates a new expense document and saves it to the MongoDB database.

**Q32. What is `_id` in MongoDB?**
> `_id` is a unique identifier automatically generated by MongoDB for every document. We use it to identify and delete specific expenses.

## 🌐 General Questions

**Q33. What is a REST API?**
> REST API is a way to communicate between frontend and backend using HTTP methods like GET (read), POST (create), DELETE (delete).

**Q34. What is the difference between GET and POST?**
> GET is used to fetch/read data. POST is used to send/create new data.

**Q35. What is `async/await`?**
> `async/await` is used to handle asynchronous operations (like database calls) in a simple, readable way without callbacks.

**Q36. What is `res.json()`?**
> It sends a JSON response back to the client from the server.

**Q37. What is the purpose of `.gitignore`?**
> It tells Git which files or folders to ignore and not push to GitHub. We ignore `node_modules/` because it is very large and can be re-installed using `npm install`.

**Q38. Why do we use `npm install` before running the project?**
> Because `node_modules` is not pushed to GitHub (it's in `.gitignore`). Running `npm install` downloads all the required packages listed in `package.json`.
